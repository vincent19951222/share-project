# P3 Polish & Acceptance Design

> 把已经落地的赛季和经济系统打磨成可内测版本，不引入新的产品主线。

## 背景

当前 P3 的核心能力已经具备：

- 服务端打卡会结算个人银子、连签、下次奖励和赛季贡献。
- 管理员可以创建赛季、结束当前赛季，并查看历史赛季。
- 看板头部和 Profile 下拉已经展示个人资产、团队金库、赛季进度和管理员入口。

问题不在核心规则，而在用户理解和试用可靠性：

- 管理员赛季页更像功能验证页，还不是一个清楚的管理面板。
- “牛马金库”“我的银子”“赛季进度”“赛季收入”之间的关系需要更明确。
- 无进行中赛季、赛季满格、重复操作、权限失败等状态需要更自然的中文反馈。
- 移动端下看板头部、Profile 下拉和管理员页面需要验收，避免内容挤压或溢出。

## 目标

1. 让管理员可以清楚地知道当前赛季状态、能否开启新赛季、如何结束当前赛季。
2. 让普通成员能理解个人银子、团队金库、连签奖励和赛季冲刺条各自代表什么。
3. 让无赛季、满格赛季、失败操作和无权限访问都有可理解的反馈。
4. 让 P3 相关界面在手机宽度下可用、可读、不会遮挡关键操作。
5. 用现有测试体系覆盖核心文案、空状态、失败状态和重要交互。

## 非目标

- 不修改打卡奖励、streak、赛季推进、撤销回滚等核心业务规则。
- 不重命名数据库字段，例如 `coins` 到 `gp`。
- 不引入团队动态、周报、企业微信推送或新的数据模型。
- 不做银子消费、商城、排行榜、成就或 Quest 系统。
- 不重做整体视觉风格，只在现有 Brutalist 语言下打磨密度、文案和响应式表现。

## 范围

### 1. 管理员赛季页

`/admin` 保持当前信息架构，但展示需要更接近正式管理面板。

需要覆盖：

- 顶部说明管理员正在管理“赛季冲刺”，不是成员账号或系统设置。
- 当前存在 active season 时，创建表单应清楚说明需要先结束当前赛季，避免提交后才报错。
- 当前赛季卡片展示：
  - 目标名称
  - 状态
  - 月份
  - 目标格数
  - 已完成格数
  - 剩余格数
  - 完成率
  - 开始时间
- 无 active season 时，展示可开启新赛季的空状态。
- 历史赛季展示：
  - 目标名称
  - 完成格数 / 目标格数
  - 完成率
  - 开始时间
  - 结束时间
  - 状态
- 刷新、创建、结束赛季的成功和失败反馈都使用中文。

### 2. 看板经济语义

看板头部继续承载高频摘要，但文案要减少歧义。

需要覆盖：

- “牛马金库”说明为团队成员个人银子的总和。
- “我的银子”说明为个人长期累计资产。
- “连签”说明为连续有效健身打卡天数。
- “下次奖励”说明为下一次有效打卡可获得的银子。
- 今日打卡统计保持简短，不增加过重说明。
- 不把赛季冲刺条描述为资产池或公共钱包。

### 3. 赛季冲刺条

`SeasonProgressBar` 保持现有方格贡献展示，但补足状态解释。

需要覆盖：

- 无 active season：说明当前没有进行中的团队冲刺，打卡仍会累计个人银子。
- active season：标题和辅助文案说明这是本期团队冲刺进度。
- 满格后：说明目标已冲满，继续打卡仍会累计个人银子和赛季收入，但进度条不再增加。
- contributor tooltip 保持贡献格数和赛季收入，不增加复杂交互。
- 移动端下 tooltip 不能成为主要信息来源，核心状态必须在常显文本里能读懂。

### 4. Profile 下拉

Profile 下拉保持轻量，不变成完整资产页。

需要覆盖：

- 展示我的银子、连签、下次奖励。
- 对“我的银子”加一句短说明，避免用户误以为它是团队公共资产。
- 管理员继续看到“赛季设置”入口。
- 移动端宽度下菜单不超出视口，按钮不被遮挡。

### 5. 错误与权限反馈

API 可以保留现有错误 code，但前端需要把常见错误翻译成中文用户反馈。

需要覆盖：

- 未登录：提示需要重新登录。
- 非管理员：提示只有管理员可以管理赛季。
- 已有 active season：提示需要先结束当前赛季。
- 无 active season 可结束：提示当前没有可结束的赛季。
- 请求体或服务端异常：提示操作没成功，并保留可重试路径。

## 设计约束

- 复用现有 `SeasonAdminPanel`、`TeamHeader`、`SeasonProgressBar`、`ProfileDropdown`。
- 复用现有 `soft-card`、粗边框、黄色按钮和深色描边语言。
- 新增 helper 函数应靠近使用处，只有多处复用时再抽到独立模块。
- 所有用户可见文案以中文为主。
- 不新增外部 UI 库。
- 保持测试运行在 Vitest + jsdom。

## 测试策略

需要补充或更新以下测试：

- `SeasonAdminPanel`：
  - active season 时显示完成率、剩余格数和开始时间。
  - active season 时创建表单给出明确提示或禁用状态。
  - 无 active season 时显示可开赛季空状态。
  - 常见 API 错误能显示中文反馈。
- `SeasonProgressBar`：
  - 无 active season 文案说明个人银子仍会累计。
  - 满格赛季显示冲满说明。
  - active season 常显文本包含目标进度。
- `ProfileDropdown`：
  - 展示个人资产说明、连签、下次奖励和管理员入口。
- 可选 CSS/组件测试：
  - 确认关键移动端 class 或结构存在，避免菜单宽度和头部布局回退。

最终验收命令：

```bash
npm test
npm run lint
```

如涉及明显布局风险，还需要启动开发服务器并做桌面与手机宽度浏览器检查。

## 验收标准

- 管理员能一眼看懂当前赛季状态，以及什么时候可以开启新赛季。
- 普通成员能从看板和 Profile 理解“我的银子”和“牛马金库”的区别。
- 无赛季、满格赛季和失败操作都有明确中文提示。
- 手机宽度下 P3 相关界面无明显遮挡、溢出或不可点击按钮。
- 不改变任何既有经济结算规则。
- 测试和类型检查通过。
