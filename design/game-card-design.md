我建议**先不改 4 个主维度**，因为它们已经很有记忆点，尤其适合“脱脂牛马 / 摸鱼办公室”这个调性。
但可以在 UI 上加一句更直白的副标题，降低理解成本：

| 维度   | 建议展示        | 定位                |
| ---- | ----------- | ----------------- |
| 把电充绿 | 站一站，不然屁股长根  | 起身、走动、拉伸、短暂恢复     |
| 把尿喝白 | 喝白白，别把自己腌入味 | 补水、接水、无糖饮品        |
| 把事办黄 | 聊两句，让班味散一散  | 闲聊、吐槽、夸夸、情绪释放     |
| 把股看红 | 看一点，给脑子补仓   | 信息输入、学习、看新闻/文章/工具 |

下面我先按**每个维度 15 张任务卡**来设计，合计 **60 张**。这个量足够支撑第一版随机轮换，也不会一上线就显得内容太薄。

# 我建议第一版卡片规则这样定

## 1. 每张卡都应该很轻

每个任务最好控制在：

* **30 秒到 5 分钟**
* 不需要拍照
* 不需要定位
* 不需要审批
* 不需要别人配合才能完成

尤其是“聊两句”维度，不能让用户觉得自己被迫社交。
如果用户当天不方便聊人，可以允许通过“发一句状态词”“写一句吐槽”“给自己说辛苦了”完成。

---

## 2. 四个维度的任务颗粒度要故意不一样

这四类不应该都像待办清单。

我建议它们的感觉分别是：

| 维度   | 用户感受             |
| ---- | ---------------- |
| 把电充绿 | 我终于从椅子上站起来了      |
| 把尿喝白 | 我终于想起来自己是个有水分的生物 |
| 把事办黄 | 我今天不是一个人在坐牢      |
| 把股看红 | 我今天脑子也稍微进账了一点    |

这样四维不会互相抢戏。

---

## 3. 完成后可以给一个“状态词”

这个很适合你们的团队动态和周报。

例如：

| 维度   | 完成状态词               |
| ---- | ------------------- |
| 把电充绿 | 已复活 / 屁股离线 / 电量+1   |
| 把尿喝白 | 已续杯 / 水位正常 / 不再干巴   |
| 把事办黄 | 班味已散 / 废话达标 / 情绪已排水 |
| 把股看红 | 信息补仓 / 脑子进账 / 今日看红  |

用户点完成时，可以默认随机一个状态词，也可以让用户改一句备注。

# 备选维度命名

如果你觉得现在这 4 个名字已经够有味儿，我建议保留。
如果想稍微更产品化一点，可以考虑这些备选：

| 当前名称 | 更克制版 | 更摸鱼版   |
| ---- | ---- | ------ |
| 把电充绿 | 身体充电 | 屁股离线   |
| 把尿喝白 | 水杯续命 | 喝到清澈   |
| 把事办黄 | 班味通风 | 废话 KPI |
| 把股看红 | 信息补仓 | 脑子看涨   |

我的推荐是：

> **主标题继续用原来的：把电充绿、把尿喝白、把事办黄、把股看红。**
> **任务卡标题用更具体、更好笑的短句。**

这样既保留了强记忆点，又不会让用户第一次看页面时完全不知道要干嘛。

# 可以直接落库的字段建议

每张任务卡建议先用这个结构：

```json
{
  "id": "charge_green_001",
  "dimension": "把电充绿",
  "title": "工位重启",
  "description": "离开椅子站起来 2 分钟，让身体退出省电模式。",
  "effort": "轻",
  "scene": "通用",
  "repeatCooldown": 3,
  "isWeekendOnly": false,
  "completionText": "电量+1"
}
```

第一版可以先不做复杂权重，只要做到：

* 每个维度每天抽 1 张；
* 最近 3 天尽量不重复；
* 工作日优先抽办公室 / 通用；
* 周末优先抽居家 / 通用；
* 用户当天允许免费换 1 次任务。
